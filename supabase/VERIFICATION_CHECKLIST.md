# Storage Bucket Verification Checklist

Use this checklist to verify the cat-images storage bucket is properly configured.

## Pre-Setup Verification

- [ ] Supabase project created
- [ ] Environment variables configured:
  - [ ] `SUPABASE_URL` set in backend/.env
  - [ ] `SUPABASE_SERVICE_KEY` set in backend/.env
  - [ ] `VITE_SUPABASE_URL` set in frontend/.env
  - [ ] `VITE_SUPABASE_ANON_KEY` set in frontend/.env

## Setup Verification

### 1. Database Migration
- [ ] Run `supabase/migration.sql` successfully
- [ ] All tables created (cat, game_run, ability)
- [ ] RLS enabled on all tables

### 2. Storage Bucket Creation
- [ ] Run `supabase/storage_setup.sql` OR `python supabase/setup_storage.py`
- [ ] Bucket `cat-images` exists
- [ ] Bucket is marked as public
- [ ] File size limit set to 10485760 bytes (10MB)
- [ ] Allowed MIME types: image/jpeg, image/png, image/webp

### 3. RLS Policies
- [ ] Policy: "Users can upload to their own folder" exists
- [ ] Policy: "Users can update their own files" exists
- [ ] Policy: "Users can delete their own files" exists
- [ ] Policy: "Public read access for all images" exists

## SQL Verification Queries

Run these in Supabase SQL Editor:

```sql
-- 1. Check bucket exists
SELECT * FROM storage.buckets WHERE id = 'cat-images';
-- Expected: 1 row with public=true, file_size_limit=10485760

-- 2. Check policies exist
SELECT policyname, cmd, qual::text 
FROM pg_policies 
WHERE schemaname = 'storage' 
  AND tablename = 'objects';
-- Expected: 4 rows (upload, update, delete, select policies)

-- 3. List uploaded files (replace with actual user_id)
SELECT name, created_at, metadata->>'size' as size_bytes
FROM storage.objects
WHERE bucket_id = 'cat-images'
  AND name LIKE 'USER_ID_HERE/%'
ORDER BY created_at DESC;

-- 4. Check storage usage
SELECT 
  bucket_id,
  COUNT(*) as file_count,
  SUM((metadata->>'size')::bigint) as total_bytes,
  pg_size_pretty(SUM((metadata->>'size')::bigint)) as total_size
FROM storage.objects
WHERE bucket_id = 'cat-images'
GROUP BY bucket_id;
```
